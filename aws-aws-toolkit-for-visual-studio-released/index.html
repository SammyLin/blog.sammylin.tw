[AWS]AWS Toolkit for Visual Studio 釋放了

| Comments

身為過去微軟的愛好者我不得不分享一下今天這個Toolkit了，過去只能使用aws toolkit for eclipse，現在多了一個AWS Toolkit for Visual Studio Amazon真的照顧廣大的人群們，哈哈~~~

名稱：AWS Toolkit for Visual Studio
網址：http://aws.amazon.com/visualstudio/
下載網址：http://d1q6xku6craafi.cloudfront.net/latest/AWSToolsAndSDKForNet.msi

因為AWS SDK for .NET當初就有了所以不多做介紹，我們就介紹新的功能

  • AWS Explorer
  • Multiple AWS Account Support
  • AWS CloudFormation Deployment

參考網址：Announcing the AWS Toolkit for Visual Studio

在這邊使用的是VS2010

 

我們安裝完AWS Toolkit之後我們打開VS2010會在「檢視」發現AWS Explorer

打開之後會在側欄跳出「AWS Exploer」，不難發現他有多帳戶管理(Multiple AWS Account Support)的功能，如果你剛好是維護大量AWS帳戶的人就不用換帳號就要key一次帳密了

點下我們可以新增一個帳戶來管理你的AWS

(如果知道怎麼取得Key的話請到[AWS][C#]Hello World for .NET來看怎麼取得)


(感覺就是很可愛~~~)

 

AWS CloudFormation Deployment

因為是在VS2010上面，不免要學一下VS2010 Auto Deployment的功能，當然AWS Toolkit for Visual Studio也可以處理，雖然不能一秒之內開主機，但是至少可以一分之內開主機並做好設定！

我們先開一個新專案(AWS Web Project) (你也可以開啟其他的專案)

如果你一開始就有新增帳戶的話，在這邊你就可以選你要建立在哪一個帳戶上

完成之後，VS用做一個範本檔出來，也會把Key 寫在Web.config裡面(感覺很不安全)

 

因為AWS Web Project 有事前做好一個頁面，我們直接Publish TO AWS CloudFormation

選擇你要放的帳戶跟地方，他這邊有提供兩個Template，我們直接選「Single Instance」就好了

雖然馬上不用5分鐘就可以建置出一個主機，但是需要用到他的CloudFormation，如果你自已建立一台Instance的話，那就沒辦法處理了。

但是如果自己建一個Project 的話，需要給別人測試一下子，其實也可以用這個方法來處理。
