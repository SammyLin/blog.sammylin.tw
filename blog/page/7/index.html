[MSSQL]清除資料表內所有資料

| Comments

TRUNCATE TABLE name

TRUNCATE TABLE 在功能上與沒有 WHERE 子句的 DELETE 陳述式相同：兩者都會移除資料表中所有的資料列，但 TRUNCATE TABLE 速度較快，且比 DELETE 使用較少的系統與交易記錄檔資源。

TRUNCATE TABLE 從資料表中移除所有的資料列，但資料表結構、條件約束、索引等等都會保留。新資料列之識別使用的計數器會重設為資料行的初始值。如果您要保留識別計數器，請改使用 DELETE。如果您要移除資料表定義及其資料，請使用 DROP TABLE 陳述式。

使用沒有 WHERE 子句的 DELETE 陳述式代替。因為 TRUNCATE TABLE 不會有記錄，它無法啟動觸發程序。

*使用後table的識別計數器會重回1,但若使用delete識別計數器不會重回1

[RDS]長期租用Reserved DB Instances

| Comments

其實On Demand DB Instances 貴到嚇死人，因為要長期經營網站，而且又要考慮備援，所以我決定租用Reserved DB Instances，雖然要一次先付一筆年費，但花費下來其實比On-Demand 還要便宜許多。

我們以Stand Deployment，Small Db Instance 來計算一年365天一天24小時，如果使用一年的話會是美金1138.8元

365*24*0.13元= 1138.8元

然而使用Reserved DB Instances的話，以相同規格的Standard Deplyment Small DB Instance來算的話，你只要先繳交一年的年費，而之後就可以用優惠的價格來使用Instance，但如果你買了10台Reserved DB Instance， 但只開了1台DB Instance，雖然只會計算1台的時間，而你10台Reserved DB Instance算是白繳年費了，而這個白繳年費是不能退費的(我有切身之痛呀~)，要小心的使用。我們來計算一下到底便宜多少。

Amazon S3 己經擁有449億個物件

| Comments

在2011年第二季結算下來，Amazon S3已經擁有了449億個物件以及在尖峰時間中每秒29萬個requests。

不管企業怎麼樣想，雲端是多麼不安全，不穩定，之前我還聽到一個笑話，還是一個總經理嘴巴講出來的他說：「雲端發展不完全，也沒有很流行。」其實這是我一個好友去面試時從那個總經理嘴巴講出來的，不然的話應該會想當面吐他槽的。

其實不是雲端不安全也不穩定，其實雲端發展已經很建全了，只是看你怎麼使用而以，而且S3可以用API把你的檔案在本地加密之後上傳了，在S3上面的檔案也就是加過密的資料，金錀你就可以留在本地端，或者是說，把你的檔案分機密、普通，可以把你普通一般重要的資料就可以放到雲端上面去。

[AWS] Ruby可以使用 AWS SDK了

| Comments

amazon又替AWS新增一個SDK了，也就是Ruby for SDK 這樣的消息，台灣的Ruby大老們應該很開心才對，就像xdite或是ihower這樣的高手們應該會想要馬上試試看效果如何，雖然我們Ruby的老爸松本行弘，跑去了Heroku當首席架構師，Heroku本身是以Ruby的雲端運算平台，是屬於PaaS架構現在有了松本行弘的加持，一定可以更加的強大。因為個人也覺得Heroku在開發是Ruby是很方便的，只要幾個指令就可以compile放到Heroku上面了。不過我們今天主要來介紹一下AWS SDK for Ruby。

Ruby 開發中心：http://aws.amazon.com/ruby/
SDK for Ruby 介紹：http://aws.amazon.com/sdkforruby/

目前SDK for Ruby 支援的服務有：

  • Elastic Compute Cloud(EC2)
  • SimpleDB
  • Simple  Email Service(SES)
  • Simple Notifcation Service(SNS)
  • Simple Queus Service(SQS)
  • Simple Storage Service(S3)

目前SDK for Ruby 支援的版本是：

Ruby 1.8.7 以上

SDK for Ruby 安裝方式

gem install aws-sdk
